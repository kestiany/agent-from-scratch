# 第一周

## day 1 思考 4 个问题：
- 1. 我的 Agent 接收什么类型的输入？
```text
答：
一个以自然语言描述的“任务请求”，该任务可能需要：
- 拆解为多个步骤
- 调用工具
- 进行多轮思考与评估

当前版本(产品假设)仅支持文本输入；
图片、语音等多模态输入被视为未来扩展能力，而非当前核心能力。
```

- 2. 我的 Agent 内部“状态”长什么样？
```text
答：
Agent 的状态由两类信息构成：

1）任务状态：
- 任务目标（objective）
- 当前计划（plan）
- 中间结果（scratchpad）
- 历史记录（history）

2）控制状态：
- 当前执行步骤
- 是否完成
- 是否需要继续
```

- 3. 我的 Agent 通过哪些步骤完成任务？
```text
答：
Agent 的核心执行流程为：

1）理解用户意图（任务澄清）
2）规划执行步骤
3）执行当前步骤
4）评估执行结果
5）反馈：
   - 对用户：输出当前进展或结果
   - 对系统：作为是否继续 / 修正计划的依据
```

- 4. 我的 Agent 在什么情况下停止？
```text
答：
Agent 在以下任一条件满足时停止运行：

1）任务被判定为已完成
2）达到最大执行轮次
3）触发安全 / 合规策略（例如敏感内容、违规请求）
```
